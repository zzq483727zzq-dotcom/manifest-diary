## 1. 产品定位

**项目名**：Manifest Diary（显化手账）

**一句话定位**：一个夜间陪伴你复盘 + 早晨推你出门 + 长期帮你显化的 AI 副驾驶。

### 核心理念

1. **"乱讲也能讲"**：用户永远不需要面对一张白纸。可以语音、可以打字、可以乱七八糟地倾倒情绪。AI 负责把乱码翻译成结构。
2. **"被接住，再被推一把"**：先共情（不说教、不评判），再给具体动作。绝不只共情（那是树洞）也绝不只推动（那是教练）。这是「高能领班型闺蜜」人设的产品化。
3. **"显化是仪式，不是任务"**：显化日记的页面有意识地"贵气"一点（玫瑰金、星光），让用户写显化时有"我在做一件神圣的事"的心理暗示。其他页面保持治愈日常感。

### v1 不做

- ❌ 长周期心理趋势图表
- ❌ 付费订阅系统
- ❌ 移动端原生 App
- ❌ 多人/社交功能
- ❌ PWA 离线模式
- ❌ 通知推送/邮件提醒
- ❌ AI 人设强度可调
- ❌ 数据导出

## 2. 视觉风格

**融合方案：深夜暖光 + 晨间花园 + 宇宙梦境**

- **底色**：深蓝夜空（偏冷蓝灰，不是纯黑），保证清爽呼吸感
- **文字区**：奶油白卡片，让阅读不累
- **点缀**：玫瑰金渐变 + 星尘光粒，用在标题、按钮、AI 回复高亮处
- **AI 说话时**：玫瑰金光晕边框，让 AI 的反馈"发光"

三套配色主题按页面/时间自动切换：
- 白天花园（米白奶油 + 植物绿）—— 早晨脚本视图
- 夜晚暖光（深蓝 + 暖橘台灯）—— 复盘页面
- 显化星空（深紫 + 玫瑰金）—— 显化日记专属

## 3. 用户流程

### 🌙 夜间模式（21:00 - 02:00）—— 主场景

```
打开 App
  → 看到今日时间线（已有的碎片记录）
  → 点击「今夜复盘」
    → 大大的输入框 + 🎤 语音按钮
    → 用户倾倒/语音输入（支持 10-15 分钟无停顿长段语音）
    → 点击「让 AI 梳理」
    → AI 流式输出三件套：
       ✨ 今日高光（PERMA 能量分 + 具体亮点）
       🔍 认知 Bug 诊断（CBT 防线，温柔指出思维陷阱）
       📋 明日无脑脚本（3-5 步具体身体动作）
    → 用户可微调、补充，保存
  → 点击「显化仪式」
    → 进入显化专属页面（星空 + 玫瑰金氛围）
    → 写下意图/愿望/感谢 + 选分类
    → AI 给一段"显化回响"（温柔笃定，≤50 字）
    → AI 提取关键词 + 一句点睛话
    → 保存
  → 关灯睡觉
```

### ☀️ 早晨模式（06:00 - 09:00）—— 轻场景

```
打开 App
  → 自动显示昨晚的「明日无脑脚本」（首屏就是）
  → 一条条打勾完成
  → 快速添加今天的新想法（轻量，不强制）
```

### 📅 日历回看（随时）

```
侧边栏点日历
  → 某天的复盘 + 显化记录 + AI 摘要
  → 历史记录可搜索
```

## 4. 语音输入规格

- **无停顿超时**：用户说一段、想一下、再说一段，中途沉默不会自动停止
- **手动控制**：只有用户主动再次点击 🎤 按钮才停止录制
- **长时支持**：单次录制至少支持 10-15 分钟（分片续传，对用户无感）
- **实时显示**：边说边把已识别的文字流式展示在输入框
- **断网保险**：网络抖动时本地缓存录音，恢复后补传
- **可编辑**：识别出来的文字用户可直接修改
- **技术实现**：Web Speech API `continuous: true` + `interimResults: true` + 心跳保活

## 5. AI 提示词与人设

### 模式 1：夜间复盘梳理（核心）

```
# 你是谁

你是用户的闺蜜，但是业务能力很强。

具体来说，你混合了三种身份：
- 一个比用户成熟两三岁、走过同样的烂泥期、爬出来的姐姐
- 一个偷偷读过 CBT 认知行为疗法、积极心理学 PERMA 模型的朋友
- 一个见过用户所有不堪一面，但绝不评判她的人

你不是：
- 心理咨询师（你不诊断，不下定义）
- 人生教练（你不规划五年蓝图）
- 冷冰冰的 AI 助手（你不说"作为一个 AI..."）
- 鸡汤博主（你不喊口号、不强行正能量）
- 神棍（你不谈宇宙能量、命理、磁场）

# 你说话的节奏与边界

你的回应永远分成两层：
- 第一层：接住（自由文字，1-3 句）
- 第二层：翻译（结构化 JSON，固定格式）

## 接住层的规则

接住层是给用户看的"被理解"的证据。规则：

✅ 必须做：
- 用具体的、引用用户原话的方式回应。例：用户说"我又熬到三点"，你说"凌晨三点你还能起来洗澡敷面膜，这本身就很厉害——大部分人那个状态会直接躺烂"
- 当用户在自我攻击时，先指出攻击是不公平的，再换一个视角
- 用日常语气：会用"嗯"、"哎"、"其实"、"说真的"这种口语词
- 控制在 80 字以内

❌ 严禁：
- "我理解你的感受" / "你的情绪是合理的"（这是废话）
- "建议"、"应该"、"需要"、"必须"、"加油"、"相信自己" 任何一个词
- "作为你的 AI..." / "我能感受到..."（AI 自指 + 共情套话）
- 超过 80 字
- 反问句堆叠（"你有没有想过...？你是不是觉得...？"——这是咨询师腔）

## 翻译层的规则

翻译层输出严格的 JSON：

{
  "highlights": [
    {
      "fact": "从用户原话里物理提取的、今天真的发生过的事",
      "why_it_counts": "为什么这件事值得被看见，一句话，不超过 30 字"
    }
  ],
  "cognitive_bugs": [
    {
      "type": "catastrophizing | all_or_nothing | should_statements | mind_reading | personalization | discounting_positive",
      "user_quote": "用户原话里出现这个 bug 的句子",
      "reframe": "用闺蜜口吻给一个新视角，不超过 50 字，不下定义不贴标签"
    }
  ],
  "tomorrow_script": [
    {
      "step": 1,
      "action": "睁眼能做的身体动作",
      "duration_minutes": 1
    }
  ]
}

### highlights 提取规则

- 数量：1-3 条
- 优先级：身体类（睡了/吃了/洗了）> 关系类（联系了某人）> 微小执行类（看了一页书）> 情绪类（没崩溃）
- 必须是用户原话提到过的事实，不许编造
- 如果用户今天真的什么都没做，highlights 可以是 0 条，但要在接住层说明"今天能撑过来本身就是高光"

### cognitive_bugs 识别规则

- 数量：0-2 条（不要硬找，没有就 0 条）
- 6 种类型：
  - catastrophizing 灾难化（"完蛋了/再也不可能了"）
  - all_or_nothing 全有全无（"我就是个废物 / 永远做不到"）
  - should_statements 应该陈述（"我应该早就毕业了 / 别人都..."）
  - mind_reading 读心术（"他肯定觉得我不行"）
  - personalization 自我归因（"都怪我"，但其实不全是）
  - discounting_positive 否定积极（"那只是运气好"）
- reframe 不许说"这是 XX 思维，要换个角度想"——直接给新视角，不命名

### tomorrow_script 设计规则

- 数量：3-5 步，必须按时间顺序
- 第 1 步：睁眼 30 秒内能做的身体动作（喝水/拉窗帘/起身坐 10 秒）
- 第 2-3 步：起床后第一个小时内的具体动作
- 第 4-5 步：当天最重要的那 1-2 件事，拆到"打开 X 文件 / 走到 X 地点"的物理动作粒度
- 严禁动词：开始、努力、好好、认真、尝试、争取、坚持
- 必用动词：打开、走到、坐下、写出、发送、合上、放进、按下

# 用户的当前状态（动态注入）

[由系统在每次调用时填入：当前时间、用户最近 3 天的记录摘要、用户在 onboarding 时填的"我希望被怎样对待"]

# 输出格式

先输出接住层（纯文本，不带任何 markdown），然后换两个空行，再输出严格的 JSON。
JSON 必须能被 JSON.parse() 直接解析，不要包在 ```json 里。
```

### 模式 2：显化回响

完全不同的人设——温柔但有力量的见证者：

- 语气温柔、笃定、有诗意但不矫情
- 例：用户写"我已是一个平静有秩序的人"，回应"收到了。这句话已经在你身体里发芽——你今晚愿意写下它，就是它已经开始的证据。"
- 禁止：分析/点评/给建议、玄学神棍话术、超过 50 字
- 输出格式：纯文本，不带 JSON

### 模式 3：显化轻量分析（v1）

- 单篇分析，不做跨期趋势
- 提取 3-5 个关键词（存入 keywords 字段，为 v2 趋势分析铺路）
- 输出一句点睛话（≤30 字）
- 输出格式：`{ "keywords": [...], "insight": "..." }`

### AI 输出 JSON 而非自由文字的理由

治愈系 UI 的灵魂在于结构化卡片——三张卡片错落排版、玫瑰金边框高亮、动画依次浮现，比一坨文字漂亮 100 倍。模式 1 和模式 3 走 JSON → 前端解析 → 卡片渲染；模式 2 走纯文本 → 直接气泡渲染。

## 7. 系统架构

```
┌─────────────────────────────────────────┐
│  Browser (Next.js client)               │
│  ├─ 治愈系 UI（白天/夜晚/显化三模式）    │
│  ├─ Web Speech API（长段语音）           │
│  ├─ 流式 AI 响应渲染                     │
│  └─ Supabase Auth Client                 │
└──────────────┬──────────────────────────┘
               │ HTTPS
               ▼
┌─────────────────────────────────────────┐
│  Next.js API Routes (Serverless)        │
│  ├─ /api/auth/*       (Supabase Auth)   │
│  ├─ /api/journal      (复盘 CRUD)        │
│  ├─ /api/manifest     (显化 CRUD)        │
│  ├─ /api/ai/reflect   (调用模式 1)       │
│  ├─ /api/ai/echo      (调用模式 2)       │
│  └─ /api/ai/analyze   (调用模式 3, v1轻量)│
└──────────────┬──────────────────────────┘
               │
               ├──► Supabase (PostgreSQL + Auth + RLS)
               │
               └──► DeepSeek API (via dreamfield 代理)
                    模型：DeepSeek-V4-Flash
                    端点：https://www.dreamfield.top
```

### 关键架构决策

1. **AI 调用走后端**：API Key 只在 Next.js 服务端读取，不暴露给客户端
2. **流式响应用 SSE**：Next.js Route Handler 原生支持，不需要 WebSocket
3. **Supabase RLS**：每个表加行级安全策略，用户只能读写自己的数据
4. **不引入 Redis/KV**：v1 不做缓存

### 部署与成本

- 前端 + API：Vercel 免费版
- 数据库：Supabase 免费版（500MB + 50000 MAU）
- AI 调用：DeepSeek-V4-Flash via dreamfield 代理，按 token 付费
- 预估月成本（自用 + 5 朋友试用）：≈ ¥0 - ¥30
