# Manifest Diary — 视觉重做设计文档 v2

> 日期：2026-06-16
> 状态：已批准
> 作者：glm-5.2 重做（v1 视觉为 deepseek 版，本次为视觉重写）

---

## 1. 背景与目标

v1 的功能层（复盘/显化/历史/AI 闭环）已跑通，但视觉是 deepseek 早期铺出的"功能化"风格，缺乏"治愈系手账"应有的氛围与仪式感。本次重做只改视觉与交互层，不动数据模型与 AI prompt。

**目标**：把项目从"能用"提升到"治愈、贵气、有仪式感"，且随时随地打开都应景。

---

## 2. 视觉锚点（已确认）

- **底色气质**：深夜书房暖光 + 远处星点（A 的暖 + B 的星融合）
- **登录页**：极简留白
- **显化页**：贵气（香槟金 + 深紫天鹅绒）
- **主点缀色**：暖光金 `#d4af37 → #f5d77a`（偏暖的真金，不是冷调，也不是之前的玫瑰金）

| 其他 | 保持暖夜白文，金只在重要时刻出现 |

## 4. 字体系统

- **标题/品牌**：衬线体（serif），字间距拉开，仪式感
- **正文**：无衬线，行高 1.7，治愈呼吸感
- **AI 说话**：衬线斜体，区别于普通正文，像有人在轻声说
- **元信息**（时间、标签）：小号 + 大字间距，低调

> 字体加载：系统衬线优先（"Songti SC", "Noto Serif SC", serif），不引外部 Google Fonts（避免之前的中转依赖问题）。

## 5. 页面设计

### 5.1 登录 / 注册页（极简留白）

- 全屏暖夜底 + 顶部一缕台灯光晕 + 稀疏星点
- 居中半透明卡片，**无边框输入框，只有底部金色细线**
- 「✦ Manifest 显化手账」衬线标题
- 「开始」按钮：金渐变 + 按下光波扩散
- 登录成功 → 渐隐过渡到首页（非硬跳）
- 注册/登录切换：底部小字链接，不抢戏

### 5.2 首页（早晨/随时脚本视图）

- 顶部时间问候（早安/午好/晚上好，按真实时间）
- "昨晚的你给今天的你留了几步路" 文案
- 脚本步骤卡片纵向排列，每张有暖光金描边
- 勾选完成：金色对勾 + 卡片轻微收拢发光
- 空状态：插画感引导（"今夜还没写，去倒出来吧"）

### 5.3 复盘页（核心 · 三件套仪式感）

**输入态**：
- 大输入框 + 语音按钮
- 「让 AI 梳理」金边按钮，按下金光扩散

**AI 回应态（关键动画）**：
1. 接住层先出现：暖光金光晕呼吸边框 + 打字光标闪烁，像有人在对面说话
2. 三件套卡片**错落浮起**：每张 delay 0.15s，从下方淡入上移
3. 高光卡、Bug 卡、脚本卡依次浮现，不是齐刷刷
4. 流式过程中接住层文字逐字显示

**编辑/保存态**：
- 脚本可编辑，编辑时金边聚焦
- 「保存今晚的复盘」按下：卡片轻微收拢 + 金光一闪，像"封存今夜"
- 保存后渐隐回首页

**错误态**：
- 不再显示干巴巴 "API error: 502"，显示真正的上游原因
- 提供「重新生成」按钮

### 5.4 显化页（贵气 · 核心仪式）

- **深紫天鹅绒底**，区别于其他页面的暖夜
- 顶部「显化仪式」衬线大字 + 拉开字间距，像请柬
- 金尘粒子**缓慢上浮**（不是满天乱闪的星），庄重
- 分类选择：金色描边胶囊，选中时金光填充
- 意图书写区：金色细描边框，写下时有被"装裱"感
- 「✨ 写下意图」金渐变按钮
- **提交瞬间**：一道金光从下往上扫过 + 流星划过
- 显化回响：金色光晕从中心晕开，衬线斜体显示
- 关键词标签：金色描边小胶囊

### 5.5 历史日历页

- 暖夜底
- 日历格子：有记录的天用暖光金小圆点标记密度
- 单日详情：复盘卡 + 显化卡纵向排列，复用卡片样式

## 6. 全局交互动效

| 元素 | 动效 |
|------|------|
| 按钮 hover | 轻微上浮 + 金边亮起 |
| 按钮 press | 光波从中心扩散 |
| 页面切换 | 渐隐过渡 0.4s |
| 卡片入场 | 从下淡入上移，错落 delay |
| 主题切换 | 背景色 0.4s 渐变 |
| AI 流式 | 打字光标 + 光晕呼吸 |
| 勾选完成 | 金色对勾 + 收拢发光 |

**节奏原则**：关键时刻有仪式（提交、保存、AI 回应），日常操作（勾选、输入）保持轻盈快速，不滥用动效。

---

## 7. 时间感知

- 问候语按真实当前时间：早安(6-11)/午好(11-14)/下午好(14-18)/晚上好(18-6)
- 复盘页标题随时间微调（深夜更轻柔、白天更明朗）
- 显化页不随时间变，永远是贵气仪式态
- **不强制昼夜主题切换**——用户随时可能复盘，底色统一暖夜，只问候语变

---

## 8. 技术实现约束

- 所有色值用 CSS 变量定义在 `globals.css`，组件只引用变量
- 动效用 Framer Motion（已装）+ CSS 关键帧
- 不引外部字体，用系统衬线
- 保留已有的主题切换基础设施（ThemeProvider），但简化为：暖夜(默认) + cosmos(显化页)
- 移动端：375px 起适配，所有动画在移动端降级（减少粒子数、关掉重光效）

---

## 9. 不做的事（YAGNI）

- ❌ 真正的昼夜自动切换底色（只问候语变，底色统一）
- ❌ 蓝色方案（已放弃）
- ❌ 用户可调主题
- ❌ 全站金色泛滥（金只在关键位置）
- ❌ 复杂插画（用排版和光效代替）

---

## 10. 验收标准

1. 登录页一眼有"被郑重迎接"的感觉，不是表单
2. 复盘页 AI 回应时，三件套错落浮现 + 接住层发光，有"被接住"的体感
3. 显化页打开有明显贵气感（金 + 深紫天鹅绒），和复盘页区分
4. 所有关键按钮按下有仪式感动效
5. 移动端不卡、不溢出
6. 暖光金贯穿全站但不过度
